# Learn: Goal Creator

> **本技能负责 AI 引导用户创建个性化学习目标并生成课程结构**

## 功能范围
- 交互式目标创建（AI 询问用户需求）
- 目标类型识别（理论型/实践型/混合型）
- 知识源分析（搜索官方文档、识别核心概念）
- 课程结构生成（理论部分按主题，实践部分按项目阶段）
- 逐模块确认机制
- 创建课程目录和文件

---

## 触发方式

**直接触发**（任意一种）：
- "创建学习目标" / "新建学习目标" / "创建目标"
- "新学习目标" / "新建目标" / "添加目标"

**间接触发**（当用户说"我要学 X"时）：

> ⚠️ **路由判断说明**：当用户说 "我要学 X" 时，系统会进行以下判断：
> - **如果 X 是现有模块名称** → 触发 learn-module-manager（开始学习模块）
> - **如果 X 是新领域/现有模块不包含** → 触发 learn-goal-creator（创建新学习目标）

**处理流程**：
1. 检查 X 是否在现有模块列表中
2. 如果是现有模块 → 调用 learn-module-manager
3. 如果是新领域 → 调用 learn-goal-creator

## AI 引导流程

```
用户: "我要学 X"
    ↓
AI: 识别目标类型（理论/实践/混合）
    ↓
AI: 交互式询问
    ├── 学习目的？（工作/兴趣/项目）
    ├── 当前知识水平？（入门/进阶/专家）
    ├── 期望学习时长？（天/周/月）
    └── 侧重点？（理论/实践/均衡）
    ↓
AI: 分析信息源
    ├── 搜索官方文档
    ├── 识别核心概念
    └── 分析技能依赖关系
    ↓
AI: 生成课程大纲（建议）
    ├── 理论部分：按知识主题划分
    └── 实践部分：按项目阶段划分
    ↓
用户: 逐模块确认
    ├── 查看每个模块的内容
    ├── 调整顺序或内容
    └── 确认后进入下一模块
    ↓
AI: 创建课程目录结构
```

## 目标类型识别

### 类型 1：理论型（📖）

**特征**：用户表达为"学会使用 X"、"了解 X"

**课程结构**：按知识主题划分

```
课程结构（理论型）：
├── 模块1：核心概念
├── 模块2：基础功能
├── 模块3：高级特性
└── 模块4：最佳实践
```

**示例**：
- 用户："我要学 OpenClaw"
- 类型：理论型
- 目标：学会使用 OpenClaw

### 类型 2：实践型（🛠️）

**特征**：用户表达为"搭建 X"、"构建 X"、"实现 X"

**课程结构**：按项目阶段划分

```
课程结构（实践型）：
├── 模块1：项目规划
├── 模块2：环境搭建
├── 模块3：核心功能实现
├── 模块4：测试与部署
└── 模块5：优化与维护
```

**示例**：
- 用户："我要搭建一个学习系统"
- 类型：实践型
- 目标：搭建学习系统

### 类型 3：混合型（🔄）

**特征**：同时包含学习和实践需求

**课程结构**：理论部分 + 实践部分

```
课程结构（混合型）：
├── 第一阶段：理论学习
│   ├── 模块1：核心概念
│   └── 模块2：基础功能
└── 第二阶段：实践项目
    ├── 模块3：项目实现
    └── 模块4：部署与优化
```

### 1. 学习目的

```
❓ 请问您学习 X 的目的是什么？

  1. 工作需要 - 用于当前/即将开始的工作
  2. 兴趣学习 - 个人兴趣，扩展知识面
```

**目的影响**：
- 工作需要 → 侧重实用功能、快速上手

- 项目需求 → 针对项目所需功能定制

### 2. 当前知识水平

```
❓ 您对 X 的了解程度？

选项：
  1. 入门 - 完全不了解，从零开始
  2. 进阶 - 有相关经验，想深入学习
  3. 专家 - 已经熟练，想研究高级特性

你的选择：[等待用户输入]
```

**水平影响**：
- 入门 → 从基础概念开始，更多解释
- 进阶 → 跳过基础，直奔高级特性
- 专家 → 聚焦源码分析、架构设计

### 3. 期望学习时长

```
❓ 您计划投入多少时间学习 X？

选项：
  1. 快速（1-3天）- 了解基本用法
  2. 中等（1-2周）- 掌握核心功能
  3. 深入（1个月+）- 全面精通

你的选择：[等待用户输入]
```

**时长影响**：
- 快速 → 只包含核心模块，快速模式
- 中等 → 完整课程，标准模式
- 深入 → 包含高级模块，完整模式 + 延伸阅读

### 4. 学习侧重点

```
❓ 您希望侧重哪方面？

选项：
  1. 理论为主 - 深入理解原理和概念
  2. 实践为主 - 动手实践，边做边学
  3. 均衡发展 - 理论与实践并重

你的选择：[等待用户输入]
```

**侧重影响**：
- 理论 → 更多文档阅读、概念讲解
- 实践 → 更多练习、项目实战
- 均衡 → 理论实践各占50%

## 知识源分析

### 信息收集

1. **搜索官方文档**
   - 查找项目官网
   - 识别官方文档链接
   - 获取 GitHub 仓库

2. **识别核心概念**
   - 从文档中提取关键术语
   - 识别主要功能模块
   - 分析技能依赖关系

3. **分析难度等级**
   - 入门门槛（前置知识要求）
   - 学习曲线（概念复杂度）
   - 实践难度（环境搭建、依赖关系）

### 分析结果示例

```markdown
## 知识源分析：OpenClaw

**官方资源**：
- 官网：https://openclaw.org
- 文档：https://docs.openclaw.org
- 仓库：github.com/OpenClaw/openclaw

**核心概念**：
1. Agent - AI 代理单元
2. Session - 会话管理
3. Skills - 技能系统
4. Hooks - 钩子机制
5. MCP Bridge - MCP 协议桥接

**前置知识**：
- 基础：Node.js、命令行操作
- 推荐：了解 Claude API、AI 编排概念

**学习曲线**：⭐⭐⭐ (进阶)
- 简单：基础命令使用
- 中等：Skills 和 Hooks 配置
- 困难：高级 Agent 协作和 MCP 集成
```

## 课程生成算法

### 理论型课程生成

```
输入：知识源分析结果
输出：课程大纲（按主题）

算法：
1. 从核心概念列表中选择主要概念
2. 按"基础 → 进阶 → 高级"排序
3. 每个概念成为一个模块
4. 每个模块包含：
   - 概念定义
   - 使用场景
   - 相关命令/配置
   - 实践示例
```

### 实践型课程生成

```
输入：项目需求分析
输出：课程大纲（按阶段）

算法：
1. 分析项目开发流程
2. 按阶段划分：规划 → 搭建 → 实现 → 测试 → 部署
3. 每个阶段成为一个模块
4. 每个模块包含：
   - 阶段目标
   - 所需技能
   - 具体任务
   - 验收标准
```

### 混合型课程生成

```
输入：理论型 + 实践型
输出：课程大纲（分阶段）

算法：
1. 前置：理论基础模块（1-2个）
2. 核心：项目实现模块（3-4个）
3. 后置：总结和优化（1个）
```

## 注意事项

1. **交互式询问**：逐步询问，一次一个问题
2. **用户确认**：创建课程前必须显示预览并等待确认
3. **知识源分析**：使用 web-search 或 mcp__web-reader__webReader 获取信息
4. **个性化适配**：根据用户水平调整内容深度
5. **逐模块确认**：允许用户调整模块内容和顺序
6. **缓存提示**：课程创建后主动提示初始化缓存
