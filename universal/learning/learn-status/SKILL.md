# Learn: Status

> **本技能为常驻技能，提供状态管理和共享机制，其他技能可引用其机制**

## 功能范围
- 查看所有模块进度概览
- 查看特定模块详细状态
- 更新模块进度（解析 checklist.md 计算百分比）
- **[NEW]** 查询当前目标状态
- **[NEW]** 查询特定目标状态
- 显示进度条和状态徽章
- **提供共享机制**：模块路径映射、非阻塞更新检查

### 非阻塞更新检查（带缓存机制，共享）

**用途**：在24小时内避免重复检查上游更新，减少网络请求

**配置文件位置**：`.claude/update-config.json`

**执行逻辑**：
1. **加载配置管理工具**：
   - 如不存在，自动初始化默认配置（24小时检查间隔）
   - 检查 `jq` 工具是否可用（不可用时降级为无缓存模式）

2. **判断是否需要网络检查**：
   - 检查距离上次检查是否超过 `intervalHours`（默认24小时）
   - **在时间窗口内**：使用缓存的版本号，跳过 `git fetch upstream`
   - **超出时间窗口**：执行网络检查并更新缓存

3. **版本比较与提醒存储**：
   - 从 CHANGELOG.md 提取本地版本
   - 从缓存或 upstream 获取上游版本
   - 如上游版本更新，存储更新提醒字符串：
   ```
   ╔════════════════════════════════════════════════════════════════╗
   ║  📢 发现新版本                                                 ║
   ║                                                                ║
   ║  当前版本: v2.0.1  |  上游版本: v2.1.0                        ║
   ║  缓存时间: 2026-03-07T10:30:00Z（24小时内不重复检查）          ║
   ║                                                                ║
   ║  建议：学习结束后运行 "同步学习计划" 获取最新功能             ║
   ╚════════════════════════════════════════════════════════════════╝
   ```

4. **显示缓存状态**（可选）：
   - 如使用缓存版本，在提醒中注明"（缓存版本）"
   - 如首次运行或配置文件不存在，显示一次性提示告知新功能

5. 在最终输出前，如存在更新提醒，则添加到结果开头

**引用方式**：其他技能在需要检查更新时，直接调用此机制，不重复实现

## 更新进度

**触发方式**（任意一种）：
- "更新进度 `<模块名>`" / "更新 `<模块名>` 的学习进度" / "同步进度 `<模块名>`"
- "记录进度 `<模块名>`" / "保存进度 `<模块名>`" / "打勾 `<模块名>`"
- "标记完成项 `<模块名>`" / "更新 `<模块名>` 状态" / "刷新进度 `<模块名>`"
- "`<模块名>` 进度更新" / "帮我更新 `<模块名>` 进度" / "`<模块名>` 学了多少"

**执行步骤**：

### 步骤 0：非阻塞更新检查（带缓存机制）

引用共享的"非阻塞更新检查"机制，详见上方"共享机制"章节

### 步骤 1：读取模块的 `checklist.md`

### 步骤 2：解析学习模式和完成项：
   - 检测已勾选的学习模式（快速/完整）
   - 统计已完成的 checkbox 数量
   - 根据学习模式计算对应的完成百分比

### 步骤 3：读取 `PROGRESS.md` 获取当前进度

### 步骤 4：生成更新预览：
   ```
   Progress: X% -> Y%
   Status: 进行中
   进度条: ██████░░░░░░░░░░░░░░ 30%
   ```

### 步骤 5：用户确认后更新 `PROGRESS.md`：
   - 更新模块的完成百分比
   - 添加学习日志条目

**进度计算规则**：
- 快速模式：只统计"📚 快速模式"章节下的 checkbox
- 完整模式：统计"🛠️ 完整模式"章节下的所有 checkbox
- 未选择模式：基于完整模式计算
