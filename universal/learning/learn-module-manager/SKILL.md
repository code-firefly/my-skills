# Learn: Module Manager

> **本技能负责管理学习生命周期，整合缓存检查，与目标系统协同工作**

## 功能范围
- 开始学习新模块（整合缓存检查、选择模式、初始化状态）
- 完成模块学习（验证完成度）
- 重置模块进度
- **[v4.0]** 检查并关联激活的学习目标
- **引用共享机制**：模块路径映射、非阻塞更新检查

---

## v4.0 目标系统集成

### 学习目标检测

在开始学习前，检查是否有激活的学习目标：

1. 读取 `.learning/goals/active-goal.md`
2. 如果有激活目标，将学习进度关联到目标
3. 如果无激活目标，提示用户创建或激活目标

### 目标进度同步

当模块进度更新时，同步到目标进度文件：

```
.learning/progress/<goal-name>-progress.md
```

# 开始学习

**触发方式**（任意一种）：
- "开始学习 `<模块名>`" / "启动 `<模块名>` 的学习" / "开始学习模块 `<模块名>`"
- "开始 `<模块名>`" / "进入 `<模块名>` 模块"
- "启动 `<模块名>`" / "开启 `<模块名>` 学习"
- "`<模块名>` 开始学习"

**执行步骤**：

### 步骤 0：非阻塞更新检查

引用 learn-status 的"非阻塞更新检查"机制：
- 如存在更新提醒，存储并在最终输出前显示
- 不阻塞学习流程，仅作为信息提醒

### 步骤 0.5：检查知识缓存（整合自 learn-tools）

1. 读取模块的 `knowledge/.metadata.json`（如果存在）

2. **情况 A：缓存存在**
   - 显示：`📦 发现本地缓存 (缓存日期: YYYY-MM-DD)`
   - 询问用户：
     - 1. 使用本地缓存（快速开始）
     - 2. 更新缓存（拉取最新资料）
     - 3. 跳过缓存（直接开始学习）

3. **情况 B：缓存不存在**
   - 显示：`⚠️  未找到知识缓存`
   - 询问用户：
     - 1. 初始化缓存（推荐，首次拉取）
     - 2. 跳过缓存（直接开始学习）

4. **用户选择后的处理**：
   - 选择使用/初始化/更新缓存：调用 learn-tools 的缓存功能
   - 选择跳过：直接进入下一步

### 步骤 1：模块名称别名解析

如果用户输入的模块名不在映射表中，检查别名映射：

| 旧名称 | 新名称 | 说明 |
|--------|--------|------|
| `claude-code-core` | `ai-tools-fundamentals` | AI 工具基础 |
| `mcp-basics` | `mcp-protocol` | MCP 协议入门 |
| `agent-sdk` | `agent-configuration` | Agent 配置与使用 |
| `mcp-advanced` | `mcp-advanced-config` | MCP 高级配置 |
| `openclaw-ecosystem` | `ai-orchestration` | AI 编排平台 |
| `everything-claude-code` | `ai-resources-research` | AI 资源研究 |
| `cc-switch` | `config-management` | 配置管理工具 |
| `spec-kit` | `spec-driven-dev` | 规范驱动开发 |
| `projects` | `practical-projects` | 综合实战项目 |

如果找到别名：
- 显示提示：`💡 注意：'{旧名称}' 已更名为 '{新名称}'，正在使用新名称继续...`
- 使用新名称继续后续流程

### 步骤 2：查找模块路径

从 learn-status 的映射表查找模块路径，优先匹配新名称。

### 步骤 3：验证模块状态

检查模块是否已开始学习（检查 checklist.md 的学习模式选择）
- 如果已开始，显示警告信息：
  ```
  ⚠️  该模块已经开始了！
  当前状态：{状态} {进度}
  是否要重新开始？(y/n)
  ```

### 步骤 4：选择学习模式

询问用户选择：
- 1. 📚 快速模式（约 3 天）- 理论为主
- 2. 🛠️ 完整模式（约 1 周）- 理论+实践

**模式说明**：
- 快速模式：适合快速了解概念，以阅读文档为主
- 完整模式：深入掌握技能，包含实践练习

### 步骤 5：显示更新预览

```
╔═══════════════════════════════════════════════════════╗
║                  开始学习预览                          ║
╠═══════════════════════════════════════════════════════╣
║ 模块：ai-tools-fundamentals                           ║
║ 优先级：P0 🔴 必学                                     ║
║ 模式：📚 快速模式（约 3 天）                           ║
║ 开始日期：2026-03-07                                   ║
╠═══════════════════════════════════════════════════════╣
║ 将要更新：                                             ║
║   ✓ 01-基础入门/ai-tools-fundamentals/checklist.md    ║
║   ✓ 01-基础入门/ai-tools-fundamentals/notes.md        ║
║   ✓ PROGRESS.md                                        ║
╠═══════════════════════════════════════════════════════╣
║ 确认开始学习？(y/n)                                    ║
╚═══════════════════════════════════════════════════════╝
```

### 步骤 6：更新文件

用户确认后，更新以下文件：

**checklist.md**：
```markdown

- [x] **快速模式**（约 3 天）- 理论为主
- [ ] **完整模式**（约 1 周）- 理论+实践

```

```markdown

| 2026-03-07 | ai-tools-fundamentals | 开始学习 | 选择快速模式 |
```

### 步骤 7：显示成功信息


# 📋 注意事项

1. **用户确认**：修改多个文件前，总是先显示预览并等待用户确认
2. **日期格式**：统一使用 YYYY-MM-DD 格式
3. **文件路径**：所有路径相对于仓库根目录
4. **错误处理**：如果文件不存在或模块未找到，给出明确的错误提示
5. **进度计算**：基于实际勾选的 checkbox 数量，不要估算
6. **别名处理**：用户使用旧名称时，友好提示并使用新名称继续
7. **缓存整合**：开始学习时主动检查知识缓存，提供选项而非强制
