# Learn: Goal Tracker

> **本技能负责管理多个学习目标的生命周期和状态切换**

## 功能范围
- 查看所有目标概览（按状态分组）
- 切换当前激活目标
- 暂停/恢复目标
- 完成目标
- 归档目标
- 聚焦视图（显示当前目标，折叠其他目标）

---

## 目标状态

| 状态 | 说明 | 目录 |
|------|------|------|
| **active** | 当前进行中的目标 | `.learning/goals/active/` |
| **paused** | 暂停的目标 | `.learning/goals/paused/` |
| **completed** | 已完成的目标 | `.learning/goals/completed/` |
| **archived** | 已归档的目标（软删除） | `.learning/goals/archived/` |

---

## 切换学习目标

**触发方式**（任意一种）：
- "切换到 `<学习目标名>`" / "切换学习目标 `<学习目标名>`" / "激活 `<学习目标名>`"
- "开始 `<学习目标名>`" / "继续 `<学习目标名>`" / "恢复 `<学习目标名>`"

**执行步骤**：

### 步骤 1：检查目标状态

从目标列表中查找目标及其当前状态

### 步骤 2：处理不同状态

| 当前状态 | 操作 |
|----------|------|
| **active** | 提示"该目标已是当前目标" |
| **paused** | 询问确认后激活 |
| **completed** | 询问"是否重新开始？" |
| **archived** | 询问"是否从归档恢复？" |

### 步骤 3：保存当前目标进度

如果当前有激活的目标，先保存其进度到对应的 progress 文件

### 步骤 4：加载新目标

1. 读取新目标的进度文件
2. 读取新目标的书签和笔记
3. 更新 `active-goal.md` 引用

### 步骤 5：显示切换结果

```
✅ 目标已切换！

🎯 新目标：learn-openclaw
📊 进度：35% ███████░░░░░░░░░░░░
📅 下一步：完成 MCP 协议基础学习

当前课程模块：
• MCP 协议入门 (进行中 60%)
• AI 编排平台 (未开始)
```

## 暂停学习目标

**触发方式**（任意一种）：
- "暂停学习目标" / "暂停当前学习目标" / "暂停学习"
- "休息一下" / "暂停一下" / "暂停"

**执行步骤**：

1. 保存当前进度
2. 将目标文件从 `active/` 移动到 `paused/`
3. 更新 `active-goal.md` 为无目标状态
4. 显示暂停信息

---

## 完成学习目标

**触发方式**（任意一种）：
- "完成学习目标" / "学习目标完成了" / "完成当前学习目标"
- "搞定学习目标" / "学习目标搞定了" / "结束学习目标"

**执行步骤**：

### 步骤 1：验证完成情况

显示未完成的模块和任务：

```
╔═══════════════════════════════════════════════════════╗
║                  目标完成验证                          ║
╠═══════════════════════════════════════════════════════╣
║ 目标：learn-openclaw                                   ║
║ 总体进度：95% ████████████████████░░                  ║
╠═══════════════════════════════════════════════════════╣
║ 未完成模块：                                           ║
║   ☐ 实战项目（15%）                                    ║
╠═══════════════════════════════════════════════════════╣
║ 选项：                                               ║
║   1. 完成所有模块后再标记                            ║
║   2. 强制标记为完成                                  ║
╚═══════════════════════════════════════════════════════╝
```

### 步骤 2：移动目标文件

将目标文件从 `active/` 移动到 `completed/`

### 步骤 3：记录完成信息

在目标文件中添加完成日期和总学习时长

### 步骤 4：清除当前目标

更新 `active-goal.md` 为无目标状态

## 归档学习目标

**触发方式**（任意一种）：
- "归档学习目标 `<学习目标名>`" / "删除学习目标 `<学习目标名>`"
- "归档 `<学习目标名>`" / "`<学习目标名>` 归档"

**执行步骤**：

1. 显示警告：归档后目标数据永久保留但不再显示在主列表
2. 询问确认（需要输入目标名）
3. 将目标文件从当前状态目录移动到 `archived/`
4. 显示归档完成信息

**注意**：归档是软删除，数据永久保留

## 输出格式规范

### 进度条
```
███░░░░░░░░ 30%
██████░░░░░ 60%
██████████░ 80%
████████████ 100%
```

### 状态徽章
- 🔵 未开始
- 🟡 进行中
- ✅ 已完成
- ⏸️ 暂停
- 📦 归档

### 目标类型标识
- 📖 理论型（学会使用 X）
- 🛠️ 实践型（搭建 X）
- 🔄 混合型（理论+实践）

---

## 注意事项

1. **状态转换规则**：
   - active → paused（暂停）
   - paused → active（恢复）
   - active → completed（完成）
   - 任何状态 → archived（归档）
   - archived → active（恢复）

2. **进度保存**：切换目标前必须保存当前目标的进度

3. **数据验证**：移动目标文件后验证数据完整性
